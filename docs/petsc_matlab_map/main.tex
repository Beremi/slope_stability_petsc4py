\chapter*{Reading Guide}
This document is split into two parts.
\begin{itemize}
\item \textbf{Mainline path} follows only the committed default 3D benchmark in \code{benchmarks/3d\_hetero\_ssr\_default/case.toml}.
\item \textbf{Annex} collects code paths that are present in the repository but inactive in that benchmark: seepage, alternative constitutive ownership modes, legacy tangent kernels, BDDC, Hypre/GAMG backends, direct SSR, limit load, and other optional continuation features.
\end{itemize}

Each chapter pair answers four questions:
\begin{itemize}
\item what the MATLAB code did in that part of the workflow,
\item where the default PETSc responsibility now lives,
\item what changed on the current parallel PMG path,
\item where to look next if the inactive or alternative code paths matter.
\end{itemize}

The reference thread throughout the mainline part is the classical 3D heterogeneous SSR case, centered on \matlabpkg{slope\_stability\_3D\_hetero\_SSR.m} and the PETSc runner \petscmod{cli/run\_3D\_hetero\_SSR\_capture.py}, with
\code{constitutive\_mode = overlap},
\code{tangent\_kernel = rows},
\code{method = indirect}, and
\code{pc\_backend = pmg\_shell}.
